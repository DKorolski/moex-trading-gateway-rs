# Stage 2B-1a — BrokerOrderId invariant hardening

Status: accepted.

Date: 2026-07-07.

## What changed

Stage 2B-1a hardens the `BrokerOrderId(String)` invariant before Stage 2B-2
starts spreading broker-order ids through runtime DTO/state paths:

- `BrokerOrderId` is no longer generated by the generic `string_id!` macro;
- its inner string is private;
- empty broker-native ids are rejected by `from_broker_native_exact`;
- legacy ALOR numeric ids still import only through
  `try_from_legacy_alor_numeric`;
- serde deserialization now uses the same non-empty invariant;
- FINAM read-only/order endpoint mappers no longer use an unchecked constructor
  for broker-provided ids;
- `docs/roadmap.md` fixes the accepted Stage 0–13 macro-roadmap.

## What did not change

- No strategy trading logic changed.
- No `HybridIntradayRuntime` behavior changed.
- No runtime state migration was applied yet.
- No command consumer was connected to real FINAM.
- No real FINAM `POST`/`DELETE` path was enabled.
- No Stop/SLTP/bracket/replace live behavior was enabled.
- No RI/RTS or USDRUBF migration was started.
- No `i64` surrogate adapter was introduced.

## Tests added or preserved

- `broker_order_id_public_constructor_cannot_create_empty`;
- `broker_order_id_deserialize_empty_string_rejected`;
- `broker_order_id_deserialize_nonempty_string_preserved_exact`;
- `legacy_alor_numeric_order_id_imports_as_decimal_string`;
- `legacy_alor_non_positive_order_id_is_rejected`;
- `broker_native_order_id_string_is_preserved_exactly`.

## Remaining live blockers

Stage 2B remains paper/mock/local only:

- runtime-live remains disabled;
- real FINAM command consumer remains disabled;
- strategy-driven real FINAM orders remain disabled;
- Stop/SLTP/bracket/replace/multi-leg live behavior remains disabled.
